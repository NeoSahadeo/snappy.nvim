# Contributors

Contributions are welcome!


## General

Pr for general fixes should be marked as such, following the standard \[feat\]
\[bug\] \[fix\] etc.

Steps to make a good pr:

1. Make each change small and divisble. Please add a good description of the
   changes and why it was done. DO NOT make a pr that has a single commit that
   is several hundred lines long

## Checks

For check pr's please include the snippet of code that did not work before your
patch so maintainers can test the changes.

Format the commit message as: `[parser] ... include language name ... `
